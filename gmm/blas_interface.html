BLAS、LAPACKまたはATLASとのインターフェース¶

密な行列のパフォーマンスを向上させるために、 `` gmm :: dense_matrix <T> ``型のいくつかの演算を `` LAPACK``（http://www.netlib.org/lapack/）または ` `` T = float、double、std :: complex <float>またはstd :: complex <double> ``に対して、 `ATLAS`（http://math-atlas.sourceforge.net/）実際、「ATLAS」に関しては、今までは特定のインターフェースは作られていないので、「ATLAS」のFortranインターフェースを使用すべきである。

このインタフェースを使うには、最初に `` GMM_USES_LAPACK``を定義してから、| gmm | files

\#define GMM_USES_LAPACK
\#include <gmm/gmm.h>

... your code

コンパイラーのコマンド行で-DGMM_USES_LAPACKを指定します。もちろん、 `` LAPACK``や `` ATLAS``ライブラリもリンクしなければなりません。例えば、標準的なlinux設定やg ++コンパイラでは、 `` LAPACK``をリンクするためのライブラリの追加は:

g++ ...  -llapack -lblas -lgfortanbegin -lgfortran

`` ATLAS``をリンクさせる:

g++ ... /usr/lib/atlas/liblapack.a /usr/lib/atlas/libblas.a -latlas -lgfortranbegin -lgfortran

ライブラリ `` libgfortranbegin``と `` libgfortran``はg ++コンパイラに固有であり、他のコンパイラでは異なるかもしれません。

この設定が機能しないかシステム管理者に問い合わせてください。

もちろん、必要に応じて別の操作をインターフェースするのは難しくありません。

次のインターフェースは、| gmm |に存在するアルゴリズムに対応していません。

`` gesvd``へのインタフェース（特異値分解）:

svd(dense_matrix<T> &X, dense_matrix<T> &U,
    dense_matrix<T> &Vt, std::vector<T> sigma);
svd(dense_matrix<std::complex<T> > &X, dense_matrix<std::complex<T> > &U,
    dense_matrix<std::complex<T> > &Vt, std::vector<T> sigma);
