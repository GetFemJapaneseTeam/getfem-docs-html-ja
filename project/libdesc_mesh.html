メッシュモジュール¶

説明¶

ライブラリのこの部分には、メッシュを格納および管理する役割があります (つまり、要素のコレクション (実際の要素) は、それぞれの面によって相互に接続されています。そのためには、要素の概念、参照の要素、メッシュの構造、節点の集合、幾何学的な変換、メッシュの境界またはサブゾーンの subpart を開発します。

GetFEM++ では、現時点では実際に有効なメッシュ機能はありません。複雑なオブジェクトのメッシュは、’ Gmsh’_ または ‘ GiD’_ などの既存のメッシャーからインポートする必要があります。メッシュの一部のインポート機能が書き込まれ、他のフォーマットに対して簡単に拡張することができます。

getfem_mesh.h ファイル内で宣言されているメッシュを表すオブジェクトで、メッシュの処理の基礎として GetFEM++ メッシュ (MESHFEM および MESHIM モジュールを参照) に応じて構造体の可能性も管理し、メッシュの進化 (要素の追加または除去など) に対応します。

ファイル¶

ファイル

説明

bgeot_convex_structure.h と bgeot_convex_structure.cc

頂点の座標を無視した要素の構造について説明します。

bgeot_mesh_structure.h と bgeot_mesh_structure.cc

節点の座標を無視したメッシュの構造について説明します。

bgeot_node_tab.h と bgeot_node_tab.cc

節点と格納節点の高速検索を可能にし、あまりにも多くの閉節点を識別する節点コンテナー。
bgeot_convex.h

頂点を持つ要素について説明します。

bgeot_convex_ref.h と bgeot_convex_ref.cc と bgeot_convex_structure.cc

参照要素について説明する。
bgeot_mesh.h

節点のコレクションを持つメッシュについて説明します (ただし、幾何学的変換の説明はありません)。

getfem_mesh_region.h と getfem_mesh_region.cc

メッシュ領域 (メッシュの境界または一部) を表すオブジェクト。

bgeot_geometric_trans.h と bgeot_geometric_trans.cc

幾何学的変換について説明します。

bgeot_geotrans_inv.h と bgeot_geotrans_inv.cc

幾何学的変換を反転するツール。

getfem_mesh.h と getfem_mesh.cc

メッシュ (幾何学的変換、メッシュの部分の一部、並列化のサポート) を完全に記述します。メッシュを調整するためのバンクアルゴリズムが含まれます。
getfem_deformable_mesh.h

変位フィールドに対してメッシュを変形し、それを復元できるオブジェクトを定義します。

getfem_mesher.h と getfem_mesher.cc

 An experimental mesher, in arbitrary dimension. To be used with care and quite slow (because of node optimization). It meshes geometries defined by some level sets.

getfem_import.h と getfem_import.cc

さまざまな形式のメッシュファイルをインポートします

getfem_regular_meshes.h と getfem_regular_meshes.cc

構造化メッシュを生成します

getfem_mesh_slicers.h と getfem_mesh_slicers.cc

スライスは、いくつかのスライス操作を適用してメッシュから構築されます (メッシュを平面で切り取り、球体と交差し、境界面を取るなど)。それらは後処理のために使用される (VTK または OpenDX への結果の出力等)。

getfem_mesh_slice.h と getfem_mesh_slice.cc

メッシュスライスを格納します。

状態¶

安定して、そんなに進化していません。

展望¶

現時点では、このモジュールは2つの部分に分割されており、異なる名前空間に分かれています。もちろん、1つの名前空間 ( getfem ) だけでモジュールを収集する方が首尾一貫しています。

ノート

ファイル bgeot_mesh.h の名前を getfem_basic_mesh.h に変更することができます。

A bibliographical review on how to efficiently store a mesh and implement the main operations (add a node, an element, deal with faces, find the neighbour elements, the isolated faces ...) would be interesting to make the mesh structure evolve.

感応アルゴリズムは、あまりにも多くのクローズ節点を識別する1つ (bgeot_node_tab) です。より多くの調査 (およびドキュメンテーション) がおそらく必要です。
