貢献する方法 / SavannahのGitリポジトリ¶

GetFEM++ は共同開発に基づいたオープンソースの有限要素ライブラリです。貢献したい場合は、プロジェクトのメンバー参加を頼むことができます。ドキュメンテーション、バグレポート、建設的なコメント、提案の変更、バグ修正、新モデルなど、あらゆる種類の貢献が歓迎されています...

貢献者は、もちろん、その変更がライブラリの適切な機能に影響を与えないよう注意し、これらの変更が下位互換性の原則に従うように注意する必要があります。

GetFEM++ 開発についてのタスクと議論のリストについては、 here を参照してください。

重要 ： 寄稿者は、自分の寄与が GetFEM++ のLGPLライセンスに基づいて配布されることを暗黙のうちに承諾しているとします。

GetFEM++ のメインリポジトリはフリーソフトウェア財団のソフトウェア鍛造所である Savannah にあります（ Savannah を参照）。 Savannah のプロジェクトのページは Getfem on Savannah です。 Getfem sources on Savannah も参照してください。

ソースを入手する方法¶

ソースだけが必要で、貢献したくないのであれば、次のコマンドを使うことができます。

git clone https://git.savannah.nongnu.org/git/getfem.git

いくつかの貢献をするつもりなら、最初のステップは、 GetFEM++ に参加することです。プロジェクト（このためには、サバンナアカウントを作成する必要があります）。また、sshキー（ git on Savannah を参照）を登録してから、次のコマンドを使用する必要があります。

git clone ssh://savannah-login@git.sv.gnu.org:/srv/git/getfem.git

貢献する方法¶

ファイルを変更する前に、 masterブランチで直接変更することはできない ため、開発ブランチ を作成する必要があります。ブランチ名は devel-name-subject の形であることが推奨されます。ここで、 name はあなたの名前またはログインであり、 subject は変更の主題です。たとえば、ブランチ名として devel-me-rewrite-fem-kernel を選択した場合、ブランチの作成方法は次の通りです。

git branch devel-me-rewrite-fem-kernel
git checkout devel-me-rewrite-fem-kernel

最初のコマンドはブランチを作成し、2番目のコマンドはブランチを作成します。その後、いくつかの変更を行う準備はほぼ完了です。変更のラベルを付けるために、連絡先の名前と電子メールを以下のコマンドで指定することができます:

git config --global user.name "Your Name Comes Here"

git config –global user.email you@yourdomain.example.com

ドキュメントの改善と誤植のためのブランチ¶

ドキュメントのみに貢献したい場合は、特定のブランチをビルドする必要はありません。この目的のために作成された fixmisspell ブランチをチェックアウトするだけです。

git checkout fixmisspell

変更をローカルにコミットする。¶

src/toto.cc と言うファイルをいくつか変更したり、新しいファイルを追加したら、 ローカルコミットは次のコマンドで行われます。

git add src/toto.cc
git commit -m "Your extensive commit message here"

この段階でコミットはあなたのローカルリポジトリで行われますが、Savannahリポジトリでは行われません。

Savannahリポジトリの変更をプッシュします¶

Savannahリポジトリに変更を転送することができます。

git push origin devel-me-rewrite-fem-kernel

もちろん devel-me-rewrite-fem-kernel はあなたのブランチの名前です。この段階で、あなたの変更は、Savannahリポジトリのブランチ devel-me-rewrite-fem-kernel に登録されます。 GetFEM++ のマスターブランチを変更することは許可されていないので、あなたの役割はここで終わります。

GetFEM++ のマスターブランチに変更をマージする場合は管理者に問い合わせてください。¶

十分なテストで変更を検証したら、管理者に GetFEM++ 変更をマージします。そのためには、それらのうちの1人に直接連絡するか、 getfem-commits@nongnu.org に “branch devel-me-rewrite-fem-kernelをマージしてください” というメッセージを電子メールで送ってください。その際には変更に関する短い説明文を添付してください。重要：デフォルトでは、ブランチを保存する必要がある場合を除き、ブランチはマージ後に削除されます。

他の貢献者が行った変更をマージする。¶

あなたは次のコマンドを実行することができます。

git pull origin master
git merge master

検証され統合された変更をマスターブランチに統合するために使用されます。マスターブランチでの変更の統合要求の前にこのコマンドを実行することをお勧めします。

有用なgitコマンド¶

git status  : status of your repository / branch

git log --follow "filepath"   : Show all the commits modifying the specified file (and follow the eventual change of name of the file).

gitk --follow filename : same as previous but with a graphical interface
