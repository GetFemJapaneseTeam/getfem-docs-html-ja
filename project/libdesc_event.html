イベント管理¶

説明¶

mesh, mesh_fem, mesh_im と model の説明は、それらの間にいくつかの依存関係があるという意味で つながっています。たとえば、要素がメッシュに対して禁じられている場合、 mesh_fem オブジェクトは反応しなければなりません。

ファイル¶

ファイル

説明

getfem_context.h と getfem_context.cc

イベントを管理するために、すべてのオブジェクトの派生元となるクラス context_dependencies を定義します。

状態¶

オブジェクトの単純な依存性に対処するための主なツールは getfem_context.h にあります。オブジェクト context_dependencies が定義されています。オブジェクトの依存関係を処理するために、オブジェクト context_dependencies はオブジェクトの親クラスである必要があります。オブジェクトに次のメソッドを追加します。

add_dependency(ct)¶

オブジェクト (親クラスとして context_dependencies を持つ必要があります) を、現在のオブジェクトが依存するオブジェクトのリストに追加します。

touch()¶

オブジェクトに何か変化があることを従属オブジェクトに示します。

context_check()¶

オブジェクトを更新するかどうかを確認します。更新する場合は、最初に依存関係リストにチェックを行い、オブジェクトの更新関数を呼び出します。(依存関係の更新関数は、現在のオブジェクトの更新関数の前に呼び出されます)。

context_valid()¶

オブジェクトがまだ有効なコンテキストを持っているかどうか、つまり、依存関係リスト内のオブジェクトがまだ存在するか表示します。

さらに、オブジェクトはメソッドを定義しなければなりません。

``void update_from_context(void) const``

コンテキストが変更された場合は context_check() の後に呼び出されます。

オブジェクト mesh に追加のシステムが存在します。個々の要素には、 mesh_fem と mesh_im の2つの呼び出しの間で変更された要素を検出するためにオブジェクトのバージョン番号があります。

展望¶

いくつかのオブジェクトのイベント管理は、注意して分析する必要があります。これは、 mesh_level_set 、 mesh_fem_level_set 、 partial_mesh_fem などのインスタンスの場合です。

イベント管理は、完全に反応するシステムであるためにまだ改善される必要があります。
