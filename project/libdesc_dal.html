Dal ライブラリ¶

説明¶

|gf|の最初期(最初のファイルは1995で書かれていました)には S.T.L. は使用できず、 ``dal`` 名前空間で定義されているコンテナはどこでも使用されていました。今、 |gf| では、S.T.L. コンテナは主に使用されます。 dal コンテナの残りの用途は、より歴史的またはこれらのコンテナの特異性によるものです。しかし、 GetFEM++ が新しいコンセプトのコンテナを開発することが目的のプロジェクトではないことは明らかです。したがって、 dal コンテナの使用は可能な限り削減することがあります。

さらに、 dal には、静的格納されたオブジェクト (有限要素法の記述、補助計算のための中間構造物) を扱うための基本的なアルゴリズムがいくつか含まれています。

ファイル¶

ファイル

説明
dal_config.h

主にヘッダーファイルとしてロードされます
dal_basic.h

可変サイズの配列コンテナ、 dal::dynamic_array<T> 。

dal_bit_vector.h と dal_bit_vector.cc

dal::dynamic_array<T> に基づく改良型ビットベクトルコンテナ。
dal_tas.h

dal::dynamic_array<T> に基づくヒープコンテナー。
dal_tree_sorted.h

dal::dynamic_array<T> に基づくバランスのとれたツリー格納された配列。

dal_static_stored_objects.h と dal_static_stored_objects.cc

いくつかのオブジェクトと依存関係を格納することができます。 GetFEM++ で多くのもの(有限要素法, 積分法, 事前計算,...)を保存するために使用します。
dal_naming_system.h

メソッド記述子に名前を関連付け、メソッド記述子を格納する汎用オブジェクト。有限要素法、積分法および幾何学的変換に使用されます。 dal::static_stored_object を使用しています。
dal_shared_ptr.h

boost::shared_ptr の簡略化されたバージョンです。

dal_singleton.h と dal_singleton.cc

OpenMP に対してスレッドセーフにされた単純なシングルトン実装 (シングルトンは、各スレッドが n 個複製されます)。

dal_backtrace.h と dal_backtrace.cc

デバッグのために、glibc トレースをダンプします。
